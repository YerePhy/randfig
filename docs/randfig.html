randfig.decorators module#

randfig.decorators.accept_cfg(fn: Callable[[Any], Any], *args: Any, **kwargs: Any) → Callable[[Mapping, Any], Any]#

This decorator is intended to use it with functions like random.randint or random.uniform in facebook-hydra configurations. It aims to return a callable that accepts a dict-like configuration and return a value. This way functions like random.randint or random.uniform can be instantiated to pass them to the formula kwarg of randfig.formula.Formula. For example:

_target_: randfig.Formula
keys: ["scanner_radius"]
formula:
  _target_: randfig.decorators.accept_cfg
  _args_:
    - _target_: hydra.utils.get_method
      path: random.uniform
    - 20
    - 30
Parameters:

  • fn – a callable.

  • args – arguments of that callable.

  • kwargs – kwargs of that callable.

Returns:

A callable that accepts just a dict-like configuration and returns fn(*args, **kwargs).

randfig.expressions.max_threshold_from_resolution(cfg: Mapping, resolution_key: str, peak: Number, is_percentage: bool = True, sigmas: Number = 2, jitter: Number | None = None) → Number#

Computes an upper threshold (upper_threshold) assuming a gaussian distibution:

FWHM_TO_STD = 1 / 2.355
lower_threshold = peak * (1 + sigmas * FWHM_TO_STD * resolution)
Parameters:

  • cfg – dict-like config having a resolution as a value.

  • resolution_key – key whose value is the gaussian resolution (FWHM).

  • peak – peak of the resolution curve.

  • sigmas – how many sigmas from the peak will be the upper threshold.

  • jitter – a uniform jitter with randfig.utils.add_uniform_jitter(). The value added is sampled from an uniform distribution with a = -jitter * peak, b = jitter * peak.

Returns:

Upper threshold.

Raises:

ValueError – if is_percentage is False but resolution > 1.

Returns:

Raises:

randfig.utils.add_uniform_jitter(value: Number, p: Number, reference: Number) → Number#

Ass jitter to value based on a fraction, p of reference:

value = value + random.uniform(-p * reference, p * reference)
Parameters:

  • value – jitted will be added to this argument.

  • p – fraction of reference that will be added to value.

  • reference – the jitter added to value is sampled from an uniform distribution with range a = -p * reference, b = p * reference.

Returns:

Jittered value.

Raises:

TypeError – if any of the arguments is not a Number instance.

class randfig.Formula(keys: Sequence[str], formula: Callable[[Mapping], Any])#

Bases: ConfigTransform

Compute a configuration value from a Callable[[Mapping], Any]. For example:

init_config = {"param_0": 1}
form = Formula(keys=["param_1"], formula = lambda cfg: 2 * cfg["param_0"])
out = form(init_config)

out: {'param_0': 1, 'param_1': 2}

Parameters:

class randfig.Save(save_dir: str | Path, filename: str, keys: Sequence[str] | None = None, sequence: int = 2, offset: int = 4)#

Bases: ConfigTransform

Saves the value of a sequence of nested keys as a YAML file, such value must be a typing.Mapping.

__init__(save_dir: str | Path, filename: str, keys: Sequence[str] | None = None, sequence: int = 2, offset: int = 4) → None#
Parameters:

  • save_dir – path to dir where the configuration will be saved as YAML.

  • filename – name of the file where the configuration will be saved. It must have ".yaml" extension.

  • sequence – ruamel.yaml parameter for YAML.indend method (avoid bad indents in lists).

  • offset – ruamel.yaml parameter for YAML.indend method (avoid bad indents in lists).

Raises:

  • ValueError – filename has a path structure, that is it has parents: configs.yaml is valid, configs/config.yaml is not valid.
